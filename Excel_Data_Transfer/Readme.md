複数のExcelデータから必要な情報を統合するためのプログラム

基本動作
・「上書きモード」では書き換えはデータに差異があった場合のみ行う(ただし入力元データが空白セルで出力先にデータがあった場合は書き換えを行わない)
・「既存データ優先モード」では書き換えは入力元にデータがあり、なおかつ出力先が空白セルの場合のみ書き換えを行う(ただしデータの差異があった場合はログには出力する)
・「確認用モード」では書き換えを行わない

ログ出力について
・「上書きモード」はデータに差異があり、書き換えを行った行のみ出力するようにする。
　そのため色付けは「入力元」にはデータが無かったが、「出力先」にはデータがあった場合(既存データを残した場合、書き換えを行っていないがあったはずのデータが消えた可能性がある為)

書き換えは差異が発生している場合のみ行うただし、
・空白セルで上書きは行わない(例として上書きモードだが入力元が空白で出力先にデータがある場合は上書きせずに出力先のデータをそのまま残す)
・上記のように、差異はあったが空白セルなので指定した動作と優先されるデータが違う場合はログに残すが反映されていない方を色付けしておく(上書きと既存優先のモードのどちらも)
・確認のみの場合は、一致した行は全てログに出力する。その中でデータに差異があった場合には色付けを行う
　ただし空白セルとは色を分けたいので、確認モードは空白セルは赤色、差異があった箇所は黄色のセルにして強調するようにする

1.UI 改良
    -左右フレーム均等縮小
    -条件リストに縦横スクロール
    -「＋条件を追加」ボタン右下固定
2.転記処理
    -縦横自動判定（照合キー列から判定）
    -複数列の転記対応
    -開始行指定・除外行指定
    -空白セルスキップ
    -上書き・既存優先・確認のみモード対応
3.ログ出力
    -デスクトップ「コピペログ」フォルダに日付時刻で作成
    -「入力元列」「出力先列」を縦横自動に合わせて出力
    -空白セル→黄色（FFF2CC）、差異→赤（F8CBAD）
4.ファイル形式対応
    -.xlsx, .xlsm, .xls, .csv に対応
5.安全仕様
    -指定シート以外は操作しない
    -空白セルの扱いはルール通り色付けで確認
